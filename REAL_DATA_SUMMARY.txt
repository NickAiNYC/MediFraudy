QUESTION: How do I activate the DOJ 11GB list and get real live data?

ANSWER: Your system is 100% ready. Here's exactly what to do:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: GET REAL DATA (Choose One Source)

Option A - CMS Medicare Data (RECOMMENDED FOR TESTING)
  • Go to: https://www.cms.gov/Research-Statistics-Data-and-Systems
  • Download: Provider Charge Data (11GB CSV)
  • Format: CSV with provider and claim info
  • Free: Yes

Option B - OIG Exclusion List (FRAUD CONVICTIONS)
  • Go to: https://oig.hhs.gov/exclusions/
  • Download: CSV format
  • Content: All excluded providers (fraud cases)
  • Free: Yes

Option C - State Medicaid Data (MOST ACCURATE)
  • Contact: Your state Medicaid office
  • Request: Provider billing/claim data
  • Format: Usually CSV or database export
  • Cost: Often free for registered users

Option D - NHCAA Database (REAL FRAUD PATTERNS)
  • Go to: https://www.nhcaa.org/
  • Member access required
  • Real investigation data

STEP 2: LOAD DATA (Pick One Method)

METHOD A - Automated Script (EASIEST)
  
  chmod +x load_real_data.sh
  ./load_real_data.sh /path/to/data.csv NY
  
  The script will:
  ✓ Check database connection
  ✓ Copy data to container
  ✓ Ask if you want to clear old data
  ✓ Load data with progress bar
  ✓ Verify success
  ✓ Show statistics
  ✓ Offer to run fraud detection

METHOD B - Docker Direct

  docker exec medicaid_backend python scripts/load_data.py \
    --file /path/to/data.csv \
    --state NY \
    --chunksize 50000

METHOD C - Direct Python (if backend running locally)

  cd backend
  python scripts/load_data.py \
    --file /path/to/data.csv \
    --state NY

FILE FORMAT REQUIREMENTS:

CSV/Parquet MUST have at least these columns (names are auto-mapped):

  PROVIDER COLUMNS:
    • NPI or provider_npi or billing_provider_npi_num
    • Name or provider_name
    • City or provider_city
    • State or provider_state
    • Zip or provider_zip (optional)
    • Facility_type or provider_type (optional)

  CLAIM COLUMNS:
    • Claim_id or claim_number
    • Beneficiary_id or member_id
    • Claim_date or service_date
    • Amount or total_amount or total_paid
    • Billing_code or hcpcs_code
    • Units (optional)

IMPORTANT: Column names don't need to match EXACTLY - the loader auto-maps
common variations. Extra columns are ignored. Missing columns default to NULL.

TROUBLESHOOTING:

  Problem: "Connection refused"
    → Solution: docker compose up -d

  Problem: "File not found"
    → Solution: Use full path: /Users/name/Downloads/data.csv

  Problem: "Out of memory"
    → Solution: docker update --memory 8g medicaid_backend
       OR: Reduce --chunksize to 10000

  Problem: "Duplicate key errors"
    → Solution: Answer 'y' when script asks to clear data

  Problem: "Database not responding"
    → Solution: Check database: docker compose logs medicaid_postgres
